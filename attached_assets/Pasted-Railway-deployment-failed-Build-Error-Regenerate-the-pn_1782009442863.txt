## Problem
This is a REPO service using the RAILPACK builder, failing at the BUILD_IMAGE stage. The build runs `pnpm install --frozen-lockfile --prefer-offline` which fails immediately with:

```
ERR_PNPM_LOCKFILE_CONFIG_MISMATCH  Cannot proceed with the frozen installation.
The current "overrides" configuration doesn't match the value found in the lockfile

Update your lockfile using "pnpm install --no-frozen-lockfile"
```

The SNAPSHOT_CODE stage succeeded. BUILD_IMAGE failed during dependency installation. The config comes from `pnpm-workspace.yaml` at the repo root.

## Diagnosis
The `pnpm-workspace.yaml` file contains an `overrides` section with platform-specific package exclusions for esbuild, lightningcss, @tailwindcss/oxide, rollup, and @expo/ngrok-bin, plus an esbuild version pin to `0.27.3` and a tsx override for `@esbuild-kit/esm-loader`. These overrides were added or modified after `pnpm-lock.yaml` was last generated, creating a mismatch. When pnpm runs with `--frozen-lockfile`, it compares the current configuration (including overrides) against what is stored in the lockfile header and refuses to proceed if they differ. The triggering commit (5534fe76) only modified `embedHandler.ts` and `ticketHandler.ts` — no changes to `pnpm-workspace.yaml` or `pnpm-lock.yaml` — confirming this is a pre-existing mismatch.

## Fix
This fix cannot be automated because it requires running `pnpm install` to regenerate the binary lockfile.

1. In the repository root (or Replit environment), run: `pnpm install --no-frozen-lockfile`
2. Commit and push the updated `pnpm-lock.yaml` to the `main` branch.

This synchronizes the lockfile with the current `overrides` configuration in `pnpm-workspace.yaml`, allowing `pnpm install --frozen-lockfile` to succeed during the Railway build.
